# 競馬場コース検証メモ

更新日: 2026-07-19

## 適用範囲

- コース形状の修正は「レース画面テスト」のコース検証モードだけで行う。
- 育成本編のレースには、検証完了・承認前のコースを接続しない。
- 画面ではスタンドを下、決勝線を下側ホームストレッチに統一する。
- 距離別スタートは公式図の発走線を登録し、ポケット発走は本線上の逆算位置で代用しない。

## 発見した既存データの問題

1. 距離別スタートを周長から逆算し、本線上へ自動配置していた。
2. ポケット・引き込み線を走行経路として保持していなかった。
3. 地方競馬の施行距離に全場共通配列を使用していた。
4. コース上辺側にスタンドと決勝線が置かれるレイアウトが混在していた。
5. 高低差の最大値は場別だったが、坂の位置は東京型の共通進行率を流用していた。

## 今後の登録単位

各 `venue + surface + layout + distance + direction` ごとに、`start`、`mergePath`、`mainPath`、`finish`、`elevationProfile` を登録する。公式図に発走線が確認できない距離は「未検証」とし、推測座標を確定データにしない。
